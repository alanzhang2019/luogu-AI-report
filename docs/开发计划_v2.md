## 0. 计划基线

### 0.1 项目现状（截至本次重写）

| 维度 | 状态 | 说明 |
|---|---|---|
| Luogu 抓取 | ✅ 完成 | `pyLuogu` 封装 + `.source_cache` 缓存 |
| 报告生成 | ✅ 完成 | MD/HTML/PDF，含能力雷达 / 大纲对标 / 错题题解 |
| 分析模块 | ✅ 完成 | `behavior_analyzer` / `syllabus_matcher` / `code_analyzer` |
| Web 后台 | ✅ 完成 | Flask + SQLite + 异步任务 + admin |
| 部署 | ✅ 完成 | Docker + `deploy.ps1` / `deploy.sh` |
| **学员档案** | ❌ 缺失 | 每次任务独立，缺 uid 维度的历史 |
| **家长端** | ❌ 缺失 | 无家长通知、无周报 |
| **账号系统** | ⚠️ 仅 admin | 学员/家长无独立账号 |
| **付费** | ❌ 缺失 | 仅 OPENAI_API_KEY 表单通道 |
| **多学员管理** | ❌ 缺失 | admin 后台无学员列表 |

### 0.2 核心约束

- **人力**：1 人（开发者本人）
- **周期**：3 个月
- **法规**：《个人信息保护法》PIPL +《未成年人保护法》—— 14 岁以下须监护人授权
- **依赖**：Luogu 反爬 + OpenAI 兼容 API
- **技术债**：admin 默认密码、weak secret、CWD 相对 DB、CSRF 缺失、PDF 同步阻塞（修复顺序见 §7）

### 0.3 计划目标

把项目从 **"高级一次性玩具"** 升级为 **"OI 教练 + 家长可视化反馈服务"**：
- 1 个真实教练带 ≥5 学员持续使用 30 天
- 家长周报打开率 ≥40%
- 付费转化 ≥1 单（验证商业可行，不设指标）

## 1. Phase 1 · 学员档案（Month 1）

> 目标：把"一次性任务"升级为"学员维度的持续档案"

### 1.1 数据模型

```sql
-- 新表
CREATE TABLE students (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  luogu_uid   TEXT UNIQUE NOT NULL,
  real_name   TEXT,                  -- 真实姓名（家长授权前为 NULL）
  school      TEXT,
  grade       TEXT,                  -- 入学年份
  is_minor    BOOLEAN NOT NULL,      -- 是否 <14 岁
  guardian_consent_at  DATETIME,     -- 家长授权时间戳
  note        TEXT,
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE student_cookies (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id  INTEGER NOT NULL,
  cookies     TEXT,                  -- 加密存储
  source      TEXT,                  -- 'admin' / 'self' / 'import'
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 改表：tasks 加 student_id
ALTER TABLE tasks ADD COLUMN student_id INTEGER REFERENCES students(id);
```

### 1.2 功能

- admin 后台：学员列表（增/删/查/导入）
- 提交测评表单：从"必填 cookies"改为"选学员 → 自动用该学员最新 cookies"
- 历史聚合：按 `student_id` 聚合最近 8 次任务
- 迁移：现有 `tasks.db` 中含 `luogu_uid` 字段的 → 自动创建对应 `students` 行（姓名/学校留空）

### 1.3 验收

- [ ] admin 能看到 ≥5 个学员列表
- [ ] 同一学员能生成 2 份报告，并自动出现在"历史"视图
- [ ] 数据迁移不回退：`python -m tests.test_migration` 全过

### 1.4 不做

- 学员自助账号（仍走 admin 录入）
- 学员登录/忘记密码
- 学校/班级管理后台

## 2. Phase 2 · 家长端 + 周报（Month 2）

> 目标：让家长**看得见**孩子的进步曲线，**收得到**周报

### 2.1 数据模型

```sql
CREATE TABLE guardians (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id      INTEGER NOT NULL,
  phone           TEXT,
  email           TEXT,
  display_name    TEXT,                -- "张同学家长"
  notify_channel  TEXT,                -- 'email' | 'sms' | 'wechat' | 'none'
  notify_token    TEXT UNIQUE,         -- URL 鉴权用，30 天过期
  notify_token_expires_at  DATETIME,
  consent_ip      TEXT,                -- PIPL 要求记录授权 IP
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE weekly_reports (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id      INTEGER NOT NULL,
  week_start      DATE,
  html_path       TEXT,
  pdf_path        TEXT,
  delivered_at    DATETIME,
  open_count      INTEGER DEFAULT 0
);
```

### 2.2 周报内容（HTML 模板，可邮件嵌入）

1. 本周通过 / 新增题数
2. 能力雷达对比上周
3. NOI 大纲对标变化（**最重要**：从"初窥算法基础"→"入门算法基础"）
4. 错题"从暴力到正解"摘要（≤3 题）
5. 教练手写备注（admin 录入）

### 2.3 调度

- 用 **APScheduler**（替代裸 cron，便于单进程部署）
- 触发：每周一 09:00（北京时间）
- 任务：扫描"过去 7 天有提交"的学员 → 生成 PDF → 邮件发送 + 写 `weekly_reports`
- 防重：同一 `(student_id, week_start)` 唯一

### 2.4 家长查看路径

- 邮件正文：HTML 摘要 + `https://<domain>/parent/<notify_token>` 链接
- 链接页面：本周报告 + 历史周报列表（无需登录）
- 鉴权：`notify_token` 一次性写入邮件，30 天过期，过期重新申请

### 2.5 验收

- [ ] 1 个家长收到 2 份周报并主动打开 ≥ 2 次
- [ ] 周报 PDF 在家长手机邮件客户端能正确显示中文（字体回退链验证）
- [ ] admin 后台能查看"周报打开次数"

### 2.6 不做

- 家长 App / 小程序
- 实时聊天
- 家长付费通道（见 Phase 3）
- 学员班级群发

## 3. Phase 3 · 付费墙 + 封闭测试（Month 3）

> 目标：验证**商业可行性**——哪怕只有 1 单付费，也算跑通闭环

### 3.1 定价与分级

| 等级 | 价格 | 包含 |
|---|---|---|
| 免费 | 0 | 单次报告、PDF 下载 |
| **学员档案版** | 10 元/月 | 历史曲线 + 6 维能力雷达 + 大纲对标 |
| **家长订阅版** | 30 元/月 | 周报 + 邮件推送 + 家长端页面 |

> 价格仅作占位，待 Phase 2 末 5 个家长访谈后调整。

### 3.2 实现方式（极简）

- 不接微信/支付宝官方 API
- admin 后台生成"激活码" → 微信收款备注"学员姓名" → 教练人工核验 → admin 录入激活码 → 学员端输入
- 订阅到期：APScheduler 每日 02:00 扫描 → 写入 `expired_at`

### 3.3 封闭测试

- 招募：**1 个真实教练**（熟人或本地 OI 群）
- 数量：5 个学员
- 周期：30 天
- 形式：教练每天用工具 1 次，家长每周收 1 次周报
- 反馈：第 7 / 15 / 30 天各 1 次 30 分钟访谈

### 3.4 验收

- [ ] 1 个真实教练承诺 30 天试用
- [ ] 至少 1 单付费（不限定金额）
- [ ] 第 30 天访谈产出"下一步做什么"决策点

### 3.5 不做

- 在线支付
- 自动续费
- 退款流程
- 发票

## 4. 技术栈修正

| 层 | 原计划 | v2 决策 | 理由 |
|---|---|---|---|
| Web 框架 | Python/FastAPI | **保留 Flask** | 已用 80%，迁移无收益 |
| 数据库 | PostgreSQL + TimescaleDB | **短期 SQLite**，>50 学员迁 PG | 学员 <50 时 SQLite 够用，避免过早优化 |
| 任务调度 | K8s CronJob | **APScheduler** | 单进程，单机部署 |
| 邮件 | SendGrid | **SMTP** | 简化依赖，1 个免费邮箱够 50 个家长 |
| 报表渲染 | 自建 | **复用 Playwright + matplotlib** | 已 100% 实现 |
| 鉴权 | RBAC | **单一 admin + 学员/家长 token** | 单人项目无 RBAC 需求 |
| 部署 | K8s | **Docker Compose 单机** | 已 100% 实现 |
| 移动端 | iOS/Android | **不做** | 家长邮件 + 浏览器够用 |
| 推荐算法 | 个性化路径 | **不做** | 缺数据 + 缺人手 |

## 5. 风险与应对

### 5.1 洛谷反爬升级（**高**）
- 现象：单 IP 短时间内多次抓取 → 403
- 应对：
  - 已有 `.source_cache`（按 uid+pid 维度）
  - Phase 1 加：每学员限频（每 6h 至多 1 次）
  - Phase 2 加：错峰调度（凌晨 02-05 抓取历史）
  - 备选：建立"教练合作账号池"（每教练贡献 1 个 luogu 账号）

### 5.2 PIPL 未成年人数据合规（**高**）
- 现象：14 岁以下学员姓名、学校、UID 进入数据库
- 应对：
  - `students.is_minor` 强制字段，未填写拒绝入库
  - `students.real_name` 在无 `guardian_consent_at` 时强制 NULL（只用 `luogu_uid` 代号）
  - 家长授权流程：邮件 → 点击确认链接 → 记录 IP/时间戳
  - 退出权：家长可邮件申请 → 7 日内硬删除

### 5.3 单人项目延期（中）
- 现象：3 个月 3 大模块，每模块 ~30 天，留 buffer = 0
- 应对：
  - 每月 1 号硬评估
  - 任意 P2 功能（见 §6）超 5 天未完成 → 直接砍，下月评估补回
  - 任意 P0 失败 ≥3 次 → 整月重排

### 5.4 AI 报告费用失控（中）
- 现象：5 学员 × 30 天 × 4 次重试 × 30 题 = 18000 次 LLM 调用
- 应对：
  - `.source_cache` 复用：同一学员同一题目不重跑
  - 周报不调用 LLM（用本地模板拼）
  - 失败时本地 7 天前报告降级（不调 LLM）

### 5.5 Web 端并发崩溃（中）
- 现象：Flask threading + Playwright sync 超过 2 并发即死锁
- 应对：
  - 加 `MAX_CONCURRENT_GENERATIONS=2` 信号量
  - 超限返回 503 + 提示"系统繁忙，请稍后"
